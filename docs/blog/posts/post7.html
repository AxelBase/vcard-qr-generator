vCard 4.0 vs 3.0: Why We Still Recommend 3.0 for QR Codes

Published: November 20, 2025

vCard 4.0 (RFC 6350) introduced new features — but broke compatibility in real-world contact sharing. Here’s why vCard 3.0 remains the safe choice.

Key vCard 4.0 Features That Break QR Use

  • KIND:individual — ignored or crashes many scanners
  • GENDER — not imported by most clients
  • ANNIVERSARY, BDAY with date-time — partial support
  • XML property — completely unsupported
  • Extended types (e.g. IMPP) — rarely handled

Tested Compatibility (2025)

  • iOS 18: Full vCard 3.0 support — Partial vCard 4.0
  • Android 15: Full 3.0 — KIND causes crash in some apps
  • Outlook 365: Full 3.0 — Ignores most 4.0 extensions
  • macOS Contacts: Full 3.0 — XML property breaks import

Why vCard 3.0 Wins for QR Codes

  • 100% compatibility across all devices
  • Smaller payload (no extra properties)
  • Faster parsing in scanner apps
  • No risk of silent failures

Recommendation

Use vCard 3.0 for all QR code contact sharing. The features added in 4.0 are not worth the compatibility risk in a medium where reliability is everything.

Future-proof isn’t about using the newest standard — it’s about using the one that actually works.
